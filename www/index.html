Lever has a policy that installation&upgrade of any program should be "flip-a-switch" rather than "climb-the-everest".

  • Initial audience: Computer game developers, Virtual reality, 3D graphics. This is the audience we attempt to cater with optimizations and design decisions. Lever doesn't attempt to replace other languages in domains where they work better.
  • Leverages RPython for implementation. Easier to improve and optimize than traditional programming languages.
  • Built-in eventloop and greenlets. Results in asyncronous programs that are as simple as what syncronous code was.
  • Customizable, extensible and hotswappable context free grammars. Allows to adjust the notation for the usecase.
  • Foreign function interface. Uses formatted declarations. Wrappers around C libraries can be reused across programming languages.
  • Multimethods with fixed arity. Avoids the gang of four pattern.
  • Lever is MIT licensed, to acknowledge the good work put into PyPy that made it possible.
